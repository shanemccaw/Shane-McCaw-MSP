Executive Summary
Microsoft Copilot for Microsoft 365 is transforming how organizations work — but only when deployed into a secure, well‑governed, and properly configured tenant. Copilot has access to everything your users can access. That means any gap in identity, permissions, governance, or data hygiene becomes a Copilot risk.

This checklist provides a comprehensive readiness framework used by organizations preparing for safe, compliant, and high‑impact Copilot adoption. It covers identity, security, data governance, SharePoint hygiene, Teams governance, licensing, organizational readiness, and rollout planning.

If your organization cannot confidently check every box, you are not ready for Copilot — yet.

Section 1 — Identity, Access & Security Readiness
Copilot depends on a secure identity foundation. If identity is weak, Copilot becomes unsafe.

Identity & Access
☐ Entra ID is the single authoritative identity source

☐ MFA enforced for all users

☐ Conditional Access policies applied (baseline + privileged)

☐ Legacy authentication fully disabled

☐ Privileged accounts separated from daily-use accounts

☐ Admin roles assigned using PIM

☐ Guest access reviewed and restricted

Device & Session Security
☐ Intune device compliance policies enforced

☐ App protection policies for mobile devices

☐ Session controls for unmanaged devices

☐ Browser session timeout policies configured

Section 2 — Data Governance & Sensitivity Labels
Copilot reads what your users can read. Data governance must be mature before rollout.

Sensitivity Labels
☐ Sensitivity labels created for all data categories

☐ Labels applied across SharePoint, OneDrive, Teams, and Exchange

☐ Mandatory labeling enabled

☐ Auto-labeling policies configured

☐ Label inheritance enabled in SharePoint libraries

Data Loss Prevention
☐ DLP policies configured for regulated data

☐ Endpoint DLP enabled

☐ Alerts monitored and reviewed

☐ Policies tested with real-world scenarios

Retention & Lifecycle
☐ Retention labels applied to critical content

☐ Retention policies configured for Teams, SharePoint, Exchange

☐ Orphaned sites and groups identified and cleaned

☐ Deleted users’ OneDrive lifecycle managed

Section 3 — SharePoint & OneDrive Hygiene
Copilot relies heavily on Microsoft Search. Poor SharePoint hygiene = poor Copilot results.

Information Architecture
☐ Sites organized by business function

☐ No “everyone has access” sites

☐ No personal data stored in shared locations

☐ No oversized “junk drawer” libraries

☐ Metadata and content types used consistently

Permissions
☐ No broken inheritance unless intentional

☐ No direct user permissions (groups only)

☐ External sharing reviewed and restricted

☐ Access reviews enabled for sensitive sites

Search Index Health
☐ All critical sites indexed

☐ No blocked libraries

☐ No legacy file share sync clutter

☐ Duplicate content removed or archived

Section 4 — Teams & Groups Governance
Teams sprawl directly impacts Copilot’s ability to retrieve relevant information.

Teams Structure
☐ Naming conventions enforced

☐ Team creation restricted or governed

☐ Private channels used intentionally

☐ Shared channels configured correctly

☐ Lifecycle policies applied

Groups Governance
☐ Group expiration policies enabled

☐ Naming policies enforced

☐ Group creation limited to approved roles

☐ Guest access reviewed regularly

Section 5 — Licensing & Technical Requirements
Copilot requires specific licensing and services to function.

Licensing
☐ Microsoft 365 E3/E5 or equivalent

☐ Copilot for Microsoft 365 licenses assigned

☐ Teams, SharePoint, OneDrive enabled

☐ Search & intelligence services active

Technical Dependencies
☐ Microsoft Search configured

☐ Graph API access not blocked

☐ Third‑party DLP not interfering with Copilot

☐ Network egress rules allow Microsoft endpoints
